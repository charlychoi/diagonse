# AI 온라인 마케팅 사전진단 사용자 설명서

> 이 설명서는 개발자가 아닌 **대표자·마케팅 담당자·컨설턴트**가 진단 결과를 올바르게 이해하고 실행하는 데 필요한 내용을 담고 있습니다.

## 1. 사전진단은 무엇을 확인하나요?

AI 온라인 마케팅 사전진단은 광고비를 쓰기 전에 회사 홈페이지가 고객과 검색엔진, AI에게 충분히 이해될 수 있는 상태인지 점검하는 도구입니다.

홈페이지 URL과 회사명을 입력하면 공개된 홈페이지 내용을 수집해 다음 질문에 답합니다.

- 첫 화면만 보고 누구에게 어떤 서비스를 제공하는지 알 수 있는가?
- 방문자가 문의·전화·예약 같은 다음 행동으로 쉽게 이동할 수 있는가?
- 광고를 시작했을 때 유입과 전환을 측정할 준비가 되어 있는가?
- 검색엔진과 AI가 회사·서비스·콘텐츠를 구조적으로 이해할 수 있는가?
- 경쟁사 홈페이지와 비교했을 때 부족한 기본 요소는 무엇인가?
- 지금 바로 고칠 일과 30·60·90일 동안 개선할 일은 무엇인가?

이 진단은 **광고 집행 전 상태를 빠르게 선별하는 사전진단**입니다. 매출, 광고 수익률, 실제 검색 순위, 상담 품질을 측정하는 성과 보고서는 아닙니다.

## 2. 진단 전에 준비할 정보

| 입력 항목 | 필수 | 설명 |
| --- | --- | --- |
| 홈페이지 URL | 필수 | 외부에서 로그인 없이 열 수 있는 회사 공식 홈페이지 주소 |
| 회사명 | 필수 | 고객이 실제로 검색하는 브랜드명 또는 상호 |
| 핵심 키워드 | 선택 | 고객이 서비스를 찾을 때 입력할 검색어. 쉼표로 여러 개 입력 |
| 업종 | 선택 | 예: 중장년 교육, 병원동행, B2B SaaS |
| 경쟁사 URL | 선택 | 알고 있는 경쟁사의 공식 홈페이지. 최대 3개 |
| 운영 채널 | 선택 | 네이버, 인스타그램, 유튜브, 광고 등 현재 운영 중인 채널 |

핵심 키워드와 업종을 입력하면 서비스 명확성과 경쟁사 적합성을 더 정확히 판단할 수 있습니다. 입력하지 않으면 홈페이지 제목·설명·본문에서 서비스 후보를 추출합니다.

## 3. 진단 실행 순서

1. 홈페이지 URL과 회사명을 입력합니다.
2. 가능하면 업종과 핵심 키워드를 추가합니다.
3. 비교하고 싶은 경쟁사를 알고 있으면 공식 홈페이지 주소를 입력합니다.
4. **AI 사전진단 시작하기**를 누릅니다.
5. 요약 점수보다 먼저 **신뢰도**, **취약 항목**, **우선 조치**를 확인합니다.
6. 즉시 실행 항목과 30·60·90일 로드맵을 담당자와 일정에 연결합니다.
7. 필요한 경우 Markdown·HTML·PDF로 저장해 공유합니다.

홈페이지와 경쟁사 수집, AI 웹 검색이 포함되므로 결과 생성에 수십 초가 걸릴 수 있습니다.

## 4. 전체 진단 로직

진단은 다음 순서로 진행됩니다.

1. **홈페이지 수집**
   공개 페이지에서 제목, 요약 설명, H1·H2, 본문, 메뉴, 문의 링크, 폼, 전화번호, 분석 태그, 구조화 데이터 등을 확인합니다.

2. **서비스·키워드 추론**
   사용자가 입력한 키워드를 우선 사용합니다. 키워드가 없으면 회사명을 제외하고 홈페이지 문구에서 서비스 후보를 추출합니다.

3. **5대 축 규칙 평가**
   수집된 신호를 기준으로 브랜드, 콘텐츠·SEO, UX·전환, 소셜·유료 미디어, 권위·AI 검색을 채점합니다.

4. **신뢰도 보정과 상한 적용**
   수집한 페이지가 적거나 핵심 정보가 부족하면 점수를 낮춰 보정합니다. 홈페이지 표면만 보는 진단이므로 최고점도 82점으로 제한합니다.

5. **세부 진단**
   첫 화면, 전환 동선, 광고 준비도, 서비스 페이지, 네이버 가이드, 로컬 SEO를 별도로 평가합니다.

6. **AI 심층 분석**
   AI를 사용할 수 있으면 웹 검색으로 경쟁사 후보와 구글 지도·지식 패널 상태를 조사하고, 개선 우선순위와 추천 문구를 제안합니다.

7. **실행 계획 생성**
   취약 항목을 바탕으로 Quick Win과 30·60·90일 로드맵을 만듭니다.

## 5. 종합 점수 계산 기준

종합 점수는 5대 축 점수의 단순 평균이 아니라 다음 비중을 적용한 가중 평균입니다.

| 진단 축 | 비중 | 주요 확인 내용 |
| --- | ---: | --- |
| 브랜드·포지셔닝 | 15% | title, 요약 설명, H1, 회사소개, 브랜드와 서비스의 연결 |
| 콘텐츠·SEO | 28% | 본문 분량, 제목 구조, canonical, 언어, 블로그, 이미지 설명 |
| UX·전환 | 22% | 모바일 대응, 메뉴, CTA, 문의·전화·예약·폼 연결 |
| 소셜·유료 미디어 | 15% | 공식 채널 연결, 공유 미리보기, 입력된 운영 채널 |
| 권위·AI 검색 | 20% | HTTPS, 구조화 데이터, 정책, 전문 콘텐츠, AI가 읽을 근거 |

가중 평균을 낸 뒤 홈페이지 수집 품질에 따라 신뢰도 보정을 적용하고, 다음과 같은 상한을 적용할 수 있습니다.

- 페이지를 수집하지 못한 경우: 최대 52점
- CTA와 문의 폼이 모두 약한 경우: 최대 62점
- 분석 태그와 구조화 데이터가 모두 약한 경우: 최대 70점
- 콘텐츠 허브가 없고 본문도 부족한 경우: 최대 68점
- 소셜 연결과 공유 미리보기가 모두 약한 경우: 최대 74점
- 모든 조건이 좋아도 URL 기반 사전진단의 최대 점수: 82점

### 등급 기준

| 등급 | 점수 | 해석 |
| --- | ---: | --- |
| A | 85점 이상 | URL 기반 진단에서는 상한 때문에 일반적으로 나오지 않음 |
| B | 70~84점 | 기본 구조가 비교적 양호하나 실측 확인 필요 |
| C | 55~69점 | 광고 전 핵심 취약 항목을 보강해야 함 |
| D | 40~54점 | 메시지·전환·측정 기반이 부족함 |
| F | 39점 이하 | 홈페이지의 기본 설명과 전환 구조부터 재정비 필요 |

점수 자체보다 **왜 그 점수가 나왔는지**, **신뢰도가 어느 정도인지**, **무엇을 먼저 고쳐야 하는지**가 더 중요합니다.

## 6. 5대 축 상세 기준

### 브랜드·포지셔닝

- 페이지 제목에 브랜드와 핵심 서비스가 드러나는지
- 검색 결과에 표시될 요약 설명이 충분한지
- H1이 하나의 핵심 메시지로 집중되어 있는지
- 회사소개·브랜드 스토리가 있는지
- 입력한 핵심 키워드가 제목·설명·헤딩에 자연스럽게 포함되는지
- HTTPS로 안전하게 접속되는지

### 콘텐츠·SEO

- canonical과 페이지 언어 설정이 있는지
- 검색과 AI가 참고할 충분한 본문이 있는지
- H2·H3로 내용이 구조화되어 있는지
- 블로그·인사이트 같은 지속적 콘텐츠 공간이 있는지
- 사이트맵 신호가 있는지
- 의미 있는 이미지에 설명 문구가 있는지

### UX·전환

- 모바일 화면에 맞게 표시되는지
- 상단 메뉴와 하단 정보가 명확한지
- 문의·상담·신청 CTA가 있는지
- CTA가 실제 전화·예약·문의 페이지로 연결되는지
- 문의 폼과 개인정보처리방침이 준비되어 있는지

### 소셜·유료 미디어

- 공식 소셜 채널이 홈페이지에 연결되어 있는지
- 링크 공유 시 제목·설명·이미지가 올바르게 표시되는지
- 사용자가 입력한 운영 채널과 홈페이지 메시지가 연결되는지

이 항목은 채널 링크와 사용자 입력을 평가합니다. 최근 게시 빈도, 광고비, 팔로워 품질은 자동 측정하지 않습니다.

### 권위·AI 검색

- Organization·LocalBusiness·FAQ 같은 구조화 데이터가 있는지
- 개인정보처리방침 등 신뢰 문서가 있는지
- 전문가·사례·가이드 콘텐츠가 충분한지
- 검색엔진과 AI가 회사와 서비스를 연결할 근거가 있는지
- 외부 참조 관계와 공식 채널 정보가 있는지

## 7. 세부 진단 항목 해석

### 첫 화면 메시지

첫 화면의 핵심 문구, 서비스·혜택 명확성, 보조 설명, CTA, 고객사·후기·인증 같은 신뢰 요소를 평가합니다. 회사명이나 추상적 슬로건만 보이면 점수가 낮아질 수 있습니다.

### 전환 동선

CTA 문구만 있는지, 실제로 전화·이메일·카카오·네이버·예약·문의 페이지로 연결되는지 구분합니다. 버튼이 보여도 작동하는 경로가 없으면 **주의**로 평가됩니다.

### 광고 집행 준비도

첫 화면 메시지, CTA, 문의 연결, GA4·GTM 측정 태그, 개인정보처리방침, 모바일 대응, 전화 링크, 완료 페이지 신호를 확인합니다. 완료 페이지와 실제 전환 이벤트는 직접 제출해 확인해야 하므로 **확인 필요**로 표시될 수 있습니다.

### 서비스·상품 페이지

홈페이지뿐 아니라 수집 가능한 서비스 페이지의 제목, H1, 설명, CTA, 문의 경로를 평가합니다. 자바스크립트로만 표시되거나 접근이 차단된 페이지는 누락될 수 있습니다.

### 경쟁사 비교

메타 설명, CTA, 문의 폼, 콘텐츠 허브, 구조화 데이터, 본문 분량을 우리 홈페이지와 비교합니다. 실제 매출, 광고 성과, 브랜드 평판을 비교하는 기능은 아닙니다.

### 네이버 검색 가이드

robots, noindex, canonical, HTTPS, 사이트맵, title, description, 모바일 대응 등 네이버 서치어드바이저 공개 가이드에 가까운 기술 신호를 점검합니다. 네이버 노출 순위를 보장하거나 실제 색인 상태를 대신 확인하지는 않습니다.

### 구글 지도·로컬 SEO

홈페이지의 상호·주소·전화번호, Organization·LocalBusiness 구조화 데이터, 지도, 영업시간 같은 준비 상태를 평가합니다. AI 웹 검색이 작동하면 회사명 검색 시 지도·지식 패널 노출 여부도 참고합니다. 결과가 불명확하면 구글 검색과 비즈니스 프로필에서 직접 확인해야 합니다.

## 8. AI가 적용된 경우

AI가 연결된 로컬 사본 또는 복제 설치본에서는 규칙 기반 진단에 다음 결과가 추가됩니다. 공개 운영 사이트에는 소유자의 개인 LLM 키가 없으므로 기본적으로 규칙 기반 결과를 제공합니다.

- 홈페이지와 업종을 해석한 종합 의견
- 영향도가 높은 개선 우선순위와 이유
- 추천 첫 화면 헤드라인·보조 설명·CTA
- 같은 서비스·고객·지역 기준의 경쟁사 공식 홈페이지 후보
- 자동 선정된 경쟁사 홈페이지의 실제 표면 신호 비교
- 구글 회사명 검색의 지도·지식 패널 상태 참고
- AI 판단에 사용된 웹 출처와 경쟁사 공식 URL

AI는 확인되지 않은 매출이나 성과를 단정하지 않도록 제한되어 있습니다. 그래도 회사명이 비슷하거나 업종 정보가 부족하면 관련성이 낮은 경쟁사를 선택할 수 있으므로 후보 이름·URL·선정 이유를 반드시 확인하세요.

## 9. AI가 적용되지 않은 경우

API 키·OAuth 로그인 누락, 사용 한도, 일시적 네트워크 오류, 응답 시간 초과, AI 응답 형식 오류가 발생하면 AI 심층 분석이 비활성화됩니다.

이 경우에도 다음 기능은 계속 작동합니다.

- 홈페이지 공개 정보 수집
- 5대 축 점수와 신뢰도 보정
- 첫 화면·전환 동선·광고 준비도 평가
- 서비스 페이지·네이버 가이드·로컬 SEO 기본 평가
- 홈페이지 문구에서 키워드 후보 추출
- Quick Win과 30·60·90일 로드맵
- 사용자가 직접 입력한 경쟁사 URL 비교

다음 결과는 제한되거나 빠질 수 있습니다.

- AI 종합 의견과 맞춤형 우선순위
- AI 추천 첫 화면 문구
- 웹 검색 기반 경쟁사 자동 선정
- 경쟁사 URL을 입력하지 않았을 때의 자동 비교
- 구글 지도·지식 패널 AI 검색 결과
- 웹 출처 인용

| 결과 항목 | AI 정상 적용 | AI 미적용 |
| --- | --- | --- |
| 5대 축 점수 | 제공 | 제공 |
| 신뢰도·점수 상한 | 적용 | 적용 |

| 키워드 후보 | 홈페이지 분석 + 설정에 따라 AI 보강 | 홈페이지 문구 기반 추출 |
| 전략 우선순위 | AI가 회사·업종 맥락을 반영 | 규칙과 취약 점수 중심 |
| 추천 문구 | 맞춤형 헤드라인·설명·CTA | 기본 Before→After 규칙안 |
| 경쟁사 자동 검색 | 최대 3개 후보 | 제공되지 않음 |
| 직접 입력한 경쟁사 비교 | 제공 | 제공 |
| 구글 패널 참고 | AI 웹 검색 결과 | 수동 확인 안내 |
| 출처 | 웹 검색 출처 표시 | 없음 |

**중요:** AI가 켜졌다고 종합 점수가 자동으로 높아지는 것은 아닙니다. 종합 점수는 주로 홈페이지에서 확인한 구조와 콘텐츠 신호로 계산합니다. AI는 해석, 웹 조사, 경쟁사 선정, 추천 전략을 보강하는 역할입니다.

## 10. 결과 상태 표시의 의미

| 상태 | 의미 |
| --- | --- |
| 양호·통과 | 홈페이지에서 해당 신호를 확인함 |
| 주의·보강 | 일부는 있으나 부족하거나 연결이 불완전함 |
| 취약·미흡 | 필요한 신호를 찾지 못했거나 명백한 문제가 있음 |
| 확인 필요·수동확인 | 홈페이지 코드만으로 확정할 수 없어 사람이 직접 확인해야 함 |

“미검출”은 반드시 “존재하지 않음”을 뜻하지 않습니다. 자바스크립트로 늦게 표시되거나, 로그인 뒤에 있거나, 수집이 차단된 정보는 자동 진단에서 보이지 않을 수 있습니다.

## 11. 결과를 실행 계획으로 바꾸는 방법

1. **취약이면서 영향도가 높은 항목**부터 고릅니다.
2. 광고를 집행 중이라면 CTA, 문의 연결, 모바일, 분석 태그를 먼저 고칩니다.
3. 검색 노출이 목표라면 title, description, H1, 서비스 페이지, FAQ, 구조화 데이터를 우선 정리합니다.
4. 로컬 사업이라면 홈페이지·구글·네이버·카카오맵의 상호·주소·전화번호를 동일하게 맞춥니다.
5. AI가 선정한 경쟁사는 공식 홈페이지와 실제 서비스가 맞는지 확인한 뒤 참고합니다.
6. 수정 후 같은 조건으로 다시 진단해 변화한 항목을 비교합니다.

점수가 오르지 않더라도 문의 경로가 실제로 짧아지거나 메시지가 명확해졌다면 중요한 개선입니다. 최종 판단은 상담 수, 전환율, 광고비 대비 성과, 검색 노출 데이터와 함께 내려야 합니다.

## 12. 진단이 측정하지 않는 것

- 네이버·구글의 실제 검색 순위와 색인 여부
- 광고비, 클릭률, 전환율, 고객획득비용
- 최근 소셜 게시 빈도와 팔로워 품질
- 로그인 이후 화면과 내부 CRM 과정
- 문의 폼을 실제 제출했을 때의 오류
- 전화 응대 품질과 상담 전환율
- 리뷰의 진위와 고객 만족도
- 경쟁사의 매출, 광고 성과, 시장점유율

이 항목들은 검색 콘솔, 광고 관리자, 분석 도구, CRM, 실제 사용자 테스트로 별도 확인해야 합니다.

## 13. 결과가 예상과 다를 때

- URL이 회사의 공식 홈페이지인지 확인합니다.
- 회사명, 업종, 핵심 키워드를 더 구체적으로 입력합니다.
- 경쟁사가 부정확하면 공식 홈페이지 URL을 직접 입력합니다.
- 홈페이지가 자바스크립트로만 구성되었다면 중요한 설명을 HTML 텍스트로도 제공하는 것이 좋습니다.
- 수집 페이지 수와 신뢰도가 낮으면 점수를 확정값으로 사용하지 않습니다.
- AI 오류가 표시되면 잠시 후 다시 실행하되 규칙 기반 결과는 그대로 활용할 수 있습니다.

## 14. 자주 묻는 질문

### 점수가 높으면 바로 광고를 시작해도 되나요?

아닙니다. 완료 페이지, 전환 이벤트, 실제 문의 제출, 모바일 사용성을 직접 확인한 뒤 집행해야 합니다.

### 점수가 낮으면 홈페이지를 전부 새로 만들어야 하나요?

반드시 그렇지는 않습니다. 보고서의 즉시 실행 항목에서 CTA, 제목, 요약 설명, 분석 태그처럼 영향도가 높고 작업량이 적은 항목부터 고치세요.

### AI가 찾은 경쟁사는 확정된 경쟁사인가요?

아닙니다. 공개 웹 정보로 찾은 후보입니다. 고객군, 서비스 범위, 지역, 가격 구조가 실제로 겹치는지 사람이 확인해야 합니다.

### 구글 지도 패널이 없다고 나오면 미등록인가요?

AI 검색에서 확인되지 않았다는 의미일 수 있습니다. 검색 위치와 계정에 따라 결과가 달라질 수 있으므로 구글 검색과 비즈니스 프로필에서 직접 최종 확인하세요.

### 이전 결과와 점수가 달라질 수 있나요?

홈페이지 내용, 수집 가능한 페이지, 입력 키워드, AI 검색 결과가 바뀌면 달라질 수 있습니다. 같은 조건으로 비교하고 변경 내역을 함께 기록하세요.
