## 1. 제품 한눈에 보기

### 무엇을 하나요

기업의 **온라인 마케팅·네이버 브랜드 검색 신호**를 홈페이지 HTML 기준으로 진단합니다.

- 단순 태그 체크리스트가 아니라  
  **「브랜드 = 핵심 서비스」 신호가 정렬되어 있는지** 를 봅니다.  
- 결과가 약하면, 네이버에서 회사명을 검색했을 때  
  “무슨 일을 하는 회사인지”가 잘 안 보일 수 있습니다.

### 어떻게 쓰나요

1. 브라우저에서 https://diagonse.vercel.app 접속  
2. **홈페이지 URL** + **회사명** 입력  
3. **진단 시작** 클릭  
4. **Markdown / HTML / PDF** 로 보고서 저장  

스킬 등록, API 키, 터미널 명령은 **일반 사용자에게 필요 없습니다.**

## 2. 시작하기 (3단계)

### ① 웹사이트 열기

브라우저 주소창에 입력:

```
https://diagonse.vercel.app
```

### ② 정보 입력

| 항목 | 필수 | 예시 |
|------|------|------|
| 회사 홈페이지 URL | ✅ | `https://sangsangwoori.com/` |
| 회사명 | ✅ | `상상우리` |
| 추가 정보 | 선택 | 키워드, 업종, 메모 |

화면의 **「예시: 상상우리」** / **「예시: 서브온」** 버튼으로 바로 시험할 수 있습니다.

### ③ 진단 실행

**진단 시작**을 누릅니다.  
홈페이지를 불러와 분석하므로 **보통 10~40초** 걸립니다.

완료되면:

- 점수 3종 카드  
- 요약 문단  
- 보고서 전문 미리보기  
- 저장 버튼 3개  

이 표시됩니다.

## 3. 입력 항목 설명

### 필수

**회사 홈페이지 URL**  
- `https://` 로 시작하는 공개 홈페이지 주소를 넣습니다.  
- 로그인해야만 보이는 페이지는 분석이 어렵습니다.

**회사명**  
- 네이버 등에서 검색될 때 쓰는 **브랜드·회사 표기**를 그대로 넣습니다.  
- 예: `상상우리`, `서브온`

### 선택 (「추가 정보 입력」 펼치기)

| 항목 | 설명 |
|------|------|
| 핵심 키워드 | 쉼표로 구분. 예: `AI 컨설팅, 중장년` |
| 업종 | 예: `AI 컨설팅·교육` |
| 메모 | 참고용 한 줄 설명 |

키워드·업종을 넣으면 브랜드=서비스 연결 판단에 도움이 됩니다.  
없어도 회사명 기준으로 진단은 진행됩니다.

## 4. 보고서 저장 (MD · HTML · PDF)

진단이 끝나면 화면 중간의 **보고서 저장** 영역을 사용합니다.

| 버튼 | 결과 | 용도 |
|------|------|------|
| **Markdown (.md)** | `.md` 파일 다운로드 | Notion, GitHub, 문서 편집 |
| **HTML (.html)** | `.html` 파일 다운로드 | 브라우저에서 열람·공유 |
| **PDF (인쇄 저장)** | 인쇄 창 오픈 | 프린터를 **「PDF로 저장」** 선택 |

### PDF 저장 방법 (상세)

1. **PDF (인쇄 저장)** 클릭  
2. 새 탭(또는 인쇄 미리보기)이 열림  
3. 대상(프린터)에서 **PDF로 저장** / **Save as PDF** / **Microsoft Print to PDF** 선택  
4. 저장  

> 팝업이 막혀 있으면 브라우저에서 이 사이트 팝업을 허용한 뒤 다시 시도하세요.

### 파일 이름 예시

```
마케팅_사전진단_상상우리_20260717_xxxxxxxx.md
마케팅_사전진단_상상우리_20260717_xxxxxxxx.html
```

## 5. 점수 이해하기

화면에 나오는 점수 **3가지는 서로 다른 의미**입니다. 한 점수로 합치지 마세요.

| 점수 | 의미 | 아닌 것 |
|------|------|---------|
| **표면 점수** | title, H1, 메타, CTA 등 HTML 표면 신호 | 실제 검색 순위 |
| **브랜드=서비스 연결** | 회사명 검색 시 “무슨 일 하는지”가 페이지에 드러나는 정도 | 광고 효율 |
| **네이버 가이드** | 서치어드바이저 기술 전제에 가까운 점검 | 네이버 노출 보장 |

- 표면 점수가 높아도 브랜드=서비스 연결이 약할 수 있습니다.  
- 본 도구는 **실측 검색 순위를 측정하지 않습니다.**

### 개선 방향 (요약)

1. **title / meta / H1** 을 `회사명 + 메인 서비스` 단위로 정렬  
   (예: 네이버 «서브온 병원동행» 검색 시 공식 홈이 연결되게)  
2. 회사명 단독 SEO만이 목표가 **아닙니다**. 메인 서비스·유관 검색어 노출이 목표입니다.  
3. 부가 사업(기업복지 등)은 메인 title에서 분리하고, 블로그·플레이스도 같은 메시지  
4. 보고서의 Before→After 문안·실검색 검증 쿼리를 따라 수정

## 7. 자주 묻는 질문

### Q. 스킬 등록이나 API 설정이 필요한가요?

**아니요.** 일반 사용자는 웹페이지만 사용하면 됩니다.

### Q. 진단이 오래 걸리거나 실패해요.

- URL이 맞는지, `https://` 가 있는지 확인  
- 사이트가 차단·지연되면 실패할 수 있습니다  
- 잠시 후 다시 시도  

### Q. PDF 버튼이 아무 반응 없어요.

- 팝업 차단을 해제  
- 또는 **HTML** 을 받은 뒤 브라우저에서 열어 **인쇄 → PDF로 저장**

### Q. 이 점수가 네이버 1페이지를 보장하나요?

**보장하지 않습니다.** HTML 신호·브랜드 연결 진단입니다.

### Q. 매뉴얼은 어디에 있나요?

https://diagonse.vercel.app/manual

### Q. 개발자 API도 있나요?

네. 같은 서버의 `POST /api/diagnose` 를 사용할 수 있습니다.  
일반 사용자 화면과 동일한 엔진입니다. 필수는 아닙니다.

```bash
curl -sS -X POST "https://diagonse.vercel.app/api/diagnose" \
  -H "Content-Type: application/json" \
  -d '{"url":"https://sangsangwoori.com/","company":"상상우리"}'
```
