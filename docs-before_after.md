# **1\. 진단 배경 — 왜 title·meta·H1인가**

검색 결과 화면에 보이는 파란 제목·회색 설명 문구는 주로 아래 네 곳에서 옵니다. 여기에 ‘서브온 \= 병원동행 전문’ 신호가 약하면, ‘서브온’ 단독 검색 결과도 빈약해 보입니다.

| 신호 원천 | 검색·사용자에게 미치는 영향 |
| ----- | ----- |
| title (문서 제목) | 검색 결과 파란 제목의 1순위 후보. 브랜드+서비스 연결의 핵심 |
| meta description | 검색 결과 회색 설명. 클릭 유도·서비스 범위 전달 |
| H1 (페이지 주제 제목) | 페이지가 ‘무엇에 대한 페이지인지’를 검색엔진에 강하게 알림 |
| 본문 앞부분 (텍스트) | 주제 보강·키워드 맥락. 이미지 속 글자는 거의 읽히지 않음 |

# **3\. Before → After 문안 초안 (즉시 반영용)**

아래 문안은 웹 소스 또는 CMS에서 교체할 수 있는 초안입니다. 확정 요금·문구는 게시 전 내부 검수(과장 표현·공공 오인 금지)를 거치십시오.

## **3-1. title**

| 구분 | 문안 |
| ----- | ----- |
| Before | 서브온 | 병원동행·기업복지 동행 서비스 |
| After 권장 A (대표안) | 서브온 | 병원동행 전문 서비스 · 부모님 병원동행 · 케어리포트 |
| After 대안 B | 서브온 병원동행 | 전문 동행매니저 · 접수부터 귀가까지 |
| After 대안 C (짧게) | 서브온 | 병원동행 전문 — 보호자 대신 함께합니다 |

* **작성 원칙:** 앞쪽에 브랜드+병원동행, ‘전문’ 명시, 기업복지는 메인 title에서 분리(기업 페이지 title로)

* **글자 수:** 검색 표시 약 30자 전후가 잘림 적음. A안은 다소 길 수 있으나 핵심 키워드 우선

* **금지:** 「서울시가 선택한」「1위」「노출 보장」 등

## **3-2. meta description**

| 구분 | 문안 |
| ----- | ----- |
| Before | 서브온은 개인 보호자와 기업 임직원 가족을 위한 병원동행·일상동행 서비스를 제공합니다. 병원 접수부터 진료, 검사, 수납, 약국, 귀가까지 함께합니다. |
| After 권장 A | 서브온은 병원동행 전문 서비스입니다. 자격 기반 동행매니저가 병원 접수·진료·검사·수납·약국·귀가까지 함께하고, 동행 후 케어리포트로 보호자에게 전달합니다. 상담 1533-1683 |
| After 대안 B | 부모님 병원 동행, 서브온에 맡기세요. 사회복지·요양·간호 자격 매니저가 안전하게 동행합니다. 누적 동행 경험과 케어리포트로 안심하세요. 지금 전화·카카오 상담 |

* **작성 원칙:** ① 병원동행 전문 한 줄 ② 무엇을 해주는지 ③ 신뢰 단서(자격·케어리포트·실적은 검증된 수치만) ④ 상담 유도

* **Before 대비 개선:** 기업 병기 축소, ‘전문’ 명시, 매니저·케어리포트·CTA 추가

* **주의:** 실적 수치(예: 25,000건)를 넣을 경우 내부 확정 수치만. 공공 수행은 「서울시 ○○사업 수행기관」 형태만, 보증 오인 금지

## **3-3. H1**

| 구분 | 문안·구조 |
| ----- | ----- |
| Before | H1 \= 로고 \+ 「보건복지형 예비사회적기업」(주제 불일치) |
| After 권장 A (대표안) | 서브온 병원동행 — 부모님 병원길, 전문 동행매니저가 함께합니다 |
| After 대안 B | 서브온 병원동행 전문 서비스 |
| After 대안 C | 병원동행이 필요할 때, 서브온 |

| 마크업 구조 After (권장) 1\) \<h1\>은 페이지 주제 한 줄만 (위 A/B/C 중 하나) 2\) 로고는 \<h1\> 밖으로: 예) \<p class="logo"\> 또는 \<div class="brand"\> \+ 이미지 3\) 「보건복지형 예비사회적기업」은 H1이 아닌 배지/본문 신뢰 영역 (span, p, li 등) 4\) 로고 alt 권장: 「서브온 — 병원동행 전문」 (영문 serve On만 쓰지 않기) |
| :---- |

## **3-4. 본문 앞부분 (히어로) — 텍스트로 올릴 문장**

이미지(visual-txt)에만 있던 메시지를 HTML 텍스트로 병기하거나 대체합니다.

| 구분 | 문안 |
| ----- | ----- |
| 히어로 헤드 | 부모님 병원 동행, 전문 매니저가 대신합니다 |
| 히어로 서브 | 접수부터 진료·검사·수납·약국·귀가까지. 동행 후 케어리포트 전달 |
| 신뢰 한 줄 | 자격 기반 동행매니저 · 보건복지형 예비사회적기업 · 상담 1533-1683 |
| CTA | 전화 상담  ·  카카오 상담  (모바일 상단 고정 권장) |

* **이미지 유지 시:** 반드시 의미 있는 alt 제공. 예) alt="서브온 병원동행 — 전문 동행매니저가 병원 전 일정을 함께합니다"

* **빈 alt 금지:** 장식 이미지만 alt="" 허용. 카피 이미지는 빈 alt 사용 금지

## **3-5. 검색 결과에 보이는 예시 (After 적용 시 기대 스니펫)**

| 가상 검색 스니펫 (보장 아님 · 방향 예시) 파란 제목: 서브온 | 병원동행 전문 서비스 · 부모님 병원동행 · 케어리포트 URL: www.theserveon.com 설명: 서브온은 병원동행 전문 서비스입니다. 자격 기반 동행매니저가 … 케어리포트로 전달합니다. 상담 1533-1683 |

# **4\. After 적용 시 SEO 최적화 방안 (실행 체크리스트)**

## **4-1. 온페이지 (메인 페이지) — 최우선**

| 순서 | 작업 | 완료 기준 |
| ----- | ----- | ----- |
| 1 | title을 After 권장안으로 교체 | 소스·브라우저 탭·검색콘솔 URL검사 |
| 2 | meta description 교체 | 소스 확인, 70\~100자 내외 핵심 전달 |
| 3 | H1을 병원동행 주제 문장으로 분리·교체 | h1 1개, 로고·배지와 분리 |
| 4 | 히어로 핵심 문장 HTML 텍스트화 \+ alt 보완 | 텍스트 선택·복사 가능 |
| 5 | 로고 alt를 한글 브랜드·서비스로 | alt에 서브온·병원동행 |
| 6 | og:title / og:description 추가 (SNS·일부 수집) | title·desc와 동일 취지 |
| 7 | 모바일 첫 화면 전화·카카오 CTA 노출 | 로그인 없이 상담 가능 경로 |

## **4-2. 콘텐츠·신뢰 신호 (메인 이하)**

* **B2C 신뢰 블록:** 매니저 자격, 선발·교육, 케어리포트, (확정 시) 누적 동행, 상담 버튼을 한 화면에

* **FAQ 8\~10개:** 전화로 자주 묻는 날짜·가격·매니저 누구·얼마나 걸리나의 답을 페이지에 텍스트로

* **기업복지:** 메인과 메시지 분리. /기업 또는 복지 전용 페이지에 별도 title·H1

* **공공 레퍼런스:** 「서울시 ○○사업 수행기관, 민간 서비스는 별도 운영」 정확한 문구. ‘서울시가 선택한’ 금지

## **4-3. 기술 SEO**

* HTML lang="ko" 유지 확인

* canonical 자기 참조 URL 설정 (중복 시)

* 모바일 반응형·속도: 히어로 이미지 압축, lazy-load

* 네이버·구글 서치콘솔에 메인 URL 재수집 요청 (수정 후)

* 구조화 데이터(Organization / LocalBusiness 등)는 별도 매뉴얼·JSON-LD 초안 참고 — 사실과 일치하는 필드만

## **4-4. 내부 링크·채널 정렬 (검색 신호 축적)**

* **동일 메시지 반복:** 홈 · 블로그 · 플레이스 · 지도 설명에 「서브온 \= 병원동행 전문」 한 방향
